JujuPets — 3 technische Ansätze

Basierend auf: Echtzeit-Sim, Pixel-Art, Aufnahme→Pflege→Vermittlung Loop, 4 Ressourcen, Matching-System, Story+Endlos

A

Pure HTML5 Canvas + Vanilla JS (Empfohlen)

Wie Game Dev Story: Ein Canvas für die Pixel-Art-Welt, DOM-Overlays für Menüs/Stats. Kein Framework-Overhead.

Vorteile

  • Volle Kontrolle über Rendering
  • Kein Build-Schritt nötig — sofort spielbar
  • Perfekt für Pixel-Art (crisp rendering)
  • Leichtgewichtig, schnell zu iterieren

Nachteile

  • Kein Ökosystem — alles selbst bauen (Game Loop, UI, State)
  • Wird bei Komplexität unübersichtlich ohne gute Architektur
B

Phaser 3 (HTML5 Game Framework)

Etabliertes 2D-Game-Framework mit Scene-Management, Sprite-System, Tween-Engine und Input-Handling.

Vorteile

  • Eingebaute Game Loop, Szenen, Sprites, Animationen
  • Große Community, viele Tutorials
  • Tilemap-Support für Tierheim-Layout
  • Plugin-System für Erweiterungen

Nachteile

  • ~1MB Framework-Größe
  • Overkill für eine Sim ohne Physik/Action
  • UI-System ist schwach — DOM-Overlay trotzdem nötig
C

DOM-basiert mit CSS Pixel-Art

Komplett HTML/CSS/JS ohne Canvas. Pixel-Art-Stil über CSS (image-rendering: pixelated, Grid-Layouts, Sprite-Sheets als Hintergründe).

Vorteile

  • Einfachste UI-Entwicklung — natürliche DOM-Elemente
  • Volle Zugänglichkeit (Accessibility)
  • Responsiv ohne extra Aufwand
  • Leicht zu debuggen (Browser DevTools)

Nachteile

  • Animationen weniger flüssig als Canvas
  • Pixel-Art-Feeling schwerer zu erreichen
  • Performance bei vielen animierten Elementen
